Over deze applicatie

Deze applicatie is gemaakt in het kader van het vak 'Clientside web frameworks' van Avans Hogeschool Breda.

AllMyGames

Bij dit project is het doel om een Angular frontend applicatie te realiseren in combinatie met een NoSQL database. Er horen 4 entiteiten bij dit project, namelijk: User, Game, Review en Developer. Het doel van mijn applicatie is dat gebruikers bij kunnen houden welke games zei en hun vrienden gespeeld hebben, nieuwe games kunnen ontdekken en games kunnen reviewen.

Entiteiten

Hier geef je een toelichting op je entiteiten. Voeg ook een afbeelding van je model toe zoals dat bij NoSql gevraagd wordt.

User

  • id: string
  • name: string
  • age: number
  • emailAdres: string
  • phoneNumber: string
  • password: string
  • friends: User[]
  • completedGames: Game[]

Game

  • id: string
  • title: string
  • summary: string
  • genre: string
  • rating: number
  • ageRating: string
  • price: number
  • completed: boolean
  • developers: Developer[]
  • reviews: Review[]

Developer

  • id: string
  • name: string
  • dateFounded: Date
  • summary: string
  • games: Game[]

Review

  • userId: User
  • gameId: Game
  • title: string
  • description: string
  • rating: number
ID Functional Requirement Acceptatiecriteria Prioriteit
F-01 Als user wil ik een lijst van games kunnen zien, zodanig dat ik weet welke games er beschikbaar zijn.
  • Er moet een lijst van games weergegeven worden.
  • Elke game toont de titel, genre, prijs, releasedatum en age rating.
  • Elke game toont een gemiddelde rating indien beschikbaar.
  • De lijst is klikbaar om naar de detailpagina te gaan.
 Must
F-02 Als user wil ik vanuit de gamelijst naar de detailpagina van een game kunnen navigeren, zodanig dat ik weet wat de game inhoudt.
  • Elke game heeft een eigen detailpagina.
  • Op de detailpagina staat een samenvatting en de bijbehorende ontwikkelaar(s).
  • Op de detailpagina worden ook alle reviews getoond.
  • Gebruikers kunnen vanaf hier een review toevoegen.
 Must
F-03 Als user wil ik een review kunnen plaatsen bij een game of developer, zodanig dat ik mijn mening kan delen.
  • De review bevat een titel, beschrijving, rating en plaatsingsdatum.
  • De review wordt gekoppeld aan een user en een game of developer.
  • Een user kan maar één review per game of developer plaatsen.
 Must
F-04 Als user wil ik mijn afgeronde games kunnen zien, zodanig dat ik mijn voortgang kan bijhouden.
  • De gebruiker heeft een profielpagina met een overzicht van voltooide games.
  • De lijst toont titels en voltooiingsdatum (indien beschikbaar).
 Should
F-05 Als user wil ik andere gebruikers kunnen toevoegen als vriend, zodanig dat ik hun voortgang en reviews kan volgen.
  • Gebruikers kunnen vriendschapsverzoeken versturen en accepteren.
  • Een gebruikersprofiel toont een lijst van vrienden.
  • Het profiel van een vriend is toegankelijk via de lijst.
